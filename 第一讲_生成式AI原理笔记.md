## 一、课程概述

本课程旨在帮助学习者理解 ChatGPT、Google Gemini、Anthropic Claude 等生成式人工智能平台的运作原理，分为原理讲解与实作两部分。

生成式人工智能（Generative AI）是能自动产生文字、图片、音乐、影片、程式码等内容的 AI 系统，核心特点是内容生成、自动创造力和数据驱动，基于深度学习模型如 GANs 或大型语言模型。

---

## 二、生成式 AI 基本原理：文字接龙的本质

### 语言模型的核心功能

所有语言模型的本质是**文字接龙**：给定未完成的句子（称为 Prompt），预测下一个最可能出现的符号（称为 Token）。例如，输入"人工智"，模型会预测下一个 Token 为"慧"；输入"大型语言模"，预测下一个 Token 为"型"。

### Token 与 Vocabulary

- **Token**：模型输出的基本单位，可能是字、词、符号等。例如中文"大家好"中，"大家"和"好"可能各为一个 Token。
- **Vocabulary（词汇表）**：模型可选择的所有 Token 集合，通常包含数十万个 Token，涵盖多语言文字、符号（如 @、emoji）等。例如 LLaMA 模型的 Vocabulary 约有 128000 个 Token。

### 概率分布与生成过程

语言模型对每个可能的 Token 赋予概率，基于概率分布"掷骰子"选择下一个 Token。例如输入"台湾最高的山是哪座？"，模型对"玉"的概率最高（可能 50%），"是"次之（可能 20%），而"冰淇淋"等荒谬 Token 的概率极低（几乎为 0）。

生成时，每次选择 Token 后会将其加入 Prompt，继续预测下一个 Token，直到输出结束符号 `[END]`。

---

## 三、语言模型的知识与能力

### 知识类型

语言模型需掌握两类知识：

- **语言知识**：理解语法规则（如"黄色的"后接名词概率高），可通过数百万篇文本学习。
- **世界知识**：掌握客观事实（如"水的沸点是摄氏 100 度"），但复杂知识（如"0.5 大气压下水的沸点"）仍有不足。

### 模型参数与训练

- **参数**：语言模型通过大量参数（通常数十亿）定义输入（Prompt）与输出（概率分布）的关系，参数通过数据自动学习，非人工设定。
- **训练数据来源**：
  - 网络爬取的海量文本（如"人工智慧真神奇"让模型学习"人→工→智"的接龙规律）
  - 人类标注（如明确"台湾最高的山？"后接"玉山"）
  - 用户反馈（通过点赞/倒赞调整答案概率，如降低不良内容的生成概率）

---

## 四、对话交互机制

### Chat Template 与 System Prompt

- **Chat Template**：平台为模型添加的对话格式，确保模型正确理解角色。例如用户输入"你是谁？"，平台会转换为"使用者问：你是谁？ AI 答："，模型仅需接续"AI 答："后的内容。不同模型的 Template 不同（如 ChatGPT 的 Template 未公开，LLaMA 有官方 Template）。
- **System Prompt**：开发者预设的隐藏指令，提供基础信息（如当前日期、模型名称）。例如添加"今天是 2025 年 9 月 12 日"，模型可正确回答日期问题。

### 多轮对话原理

多轮对话通过保留历史记录实现：将前序对话（如"使用者问：台湾最高的山是哪座？ AI 答：玉山"）与新问题（"第二高的呢？"）组合为新 Prompt，模型基于上下文理解"第二高"指"第二高的山"，从而生成"雪山"。

## 五、AI 幻觉与解决方法

### 幻觉现象

模型可能生成虚假信息（如编造不存在的网址），原因是模型无实时数据库，仅通过文字接龙生成内容。例如问"台湾大专院校人工智能学程联盟官网"，模型可能输出虚构网址。

### 解决方法：RAG 技术

通过**检索增强生成（RAG，Retrieval-Augmented Generation）**结合搜索引擎，让模型生成时参考外部真实数据，减少幻觉。例如 ChatGPT 预设启用 RAG，关闭后易产生错误答案。

---

## 六、多模态生成：图像与声音的 Token 化

生成图片、声音的原理与文字相同，核心是将内容转换为 Token 后进行接龙。

### 图像生成

先通过图像编码器将图片压缩为 16×16 的 Token（代表"草地""眼睛"等 Pattern），语言模型生成新 Token 后，经解码器还原为图片。

### 声音生成

通过声音编码器将音频转换为代表 0.02 秒声音的 Token（如"b""笑声"），模型生成新 Token 后，经解码器合成声音。

> NVIDIA CEO 黄仁勋在 2024 年 COMPUTEX 演讲中提到，Token 可代表文字、图像、图表、歌曲、语音、视频等，生成式 AI 的核心是将万物表示为 Token 并进行接龙。

## 七、开源模型实作（基于 Colab 与 Hugging Face）

### 开源模型与平台

- **开源模型**：如 Meta 的 LLaMA、Mistral、Google 的 Gemma，参数公开可下载；非开源模型（如 ChatGPT、Gemini）仅提供 API 或网页交互。
- **Hugging Face**：开源模型平台，类似"模型的 Facebook"，可查看模型说明、下载次数及评分。

### 实作步骤（以 LLaMA 3.2-3B-Instruct 为例）

1. **环境准备**：使用 Colab 平台，选择 GPU（如 T4），安装 Hugging Face Transformers 套件。

2. **获取模型**：从 Hugging Face 下载模型（需申请权限），包含 tokenizer（存储 Vocabulary）和 model（存储参数）。

3. **Token 操作**：
   - 查看 Token 总数（如 LLaMA 为 128000）
   - 将 Token 编号转换为文字（如编号 0 对应"!"）
   - 将文字转换为 Token 编号（如"大家"对应 109429）

4. **文字接龙实现**：
   - 基础方法：输入 Prompt → 转换为 Token 编号 → 模型预测下一个 Token 概率分布 → 选择 Token 并追加到 Prompt，循环至结束符号
   - 优化工具：可设置生成长度、是否随机采样、仅保留概率前 k 个 Token（Top-k）等参数

5. **对话模板应用**：通过添加官方 Chat Template，包含 system、user、assistant 角色，实现多轮对话。例如设置 system prompt"你的名字是 Llama"，模型会正确回应身份。

---

## 八、关键结论

生成式 AI 的核心是 **Token 接龙**：将复杂内容（文字、图像、声音等）分解为有限 Token，通过预测下一个 Token 的概率分布实现生成。语言模型的能力依赖于参数规模、训练数据质量及对话模板设计，而 RAG 等技术可有效提升输出准确性。实作中，开源模型（如 LLaMA、Gemma）为学习提供了可操作的工具，帮助深入理解生成式 AI 的底层逻辑。
